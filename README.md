# Website Design Skills

Five AI skill files that force AI coding tools to produce high-end, visually polished frontend code — instead of the same generic layouts, default fonts, and bland components they normally spit out.

Created by **Zara Hunter** — [Eduk8agentic](https://www.eduk8agentic.com)

## What's Inside

Each skill lives in its own folder with a single `SKILL.md` file. Pick the ones that fit your project.

### 1. design-skill
The foundation. Controls how the AI approaches layout, typography, colour, spacing, and motion from scratch. Includes configurable dials for variance, animation intensity, and visual density so you can tune the output to match the project.

### 2. upgrade-skill
For when you already have a codebase and want to make it look better without starting over. Runs a structured audit across typography, colour, layout, interactivity, and accessibility — then applies targeted fixes in priority order.

### 3. premium-skill
Pushes the AI toward agency-level output. Layered card architecture, spring-based motion, floating navigation, and strict anti-pattern enforcement. Everything that separates a polished product from a default template.

### 4. complete-skill
Prevents the AI from being lazy. Bans placeholder comments, truncated imports, skeleton outputs, and "the rest follows the same pattern" shortcuts. If you asked for a full file, you get a full file.

### 5. editorial-skill
For minimal, document-style interfaces — think Notion, Linear, or premium editorial layouts. Monochrome palettes, serif/sans-serif typographic contrast, hairline borders, generous whitespace, and muted pastel accents. No gradients, no heavy shadows.

## Getting Started

1. Grab the `SKILL.md` from whichever skill folder you want.
2. Drop it into your project root (or wherever your AI tool reads context files).
3. Reference it in your prompt or editor — for example, `@SKILL.md` in Cursor, or paste it into your system instructions.

The AI picks up the rules automatically and applies them to everything it generates.

## Tuning the Design Skill

The design skill has three dials at the top of the file. Set each one from 1 to 10 depending on what you're building:

**DESIGN_VARIANCE** — Layout experimentation level.
- 1-3: Symmetrical, centred, predictable grids.
- 4-7: Mixed aspect ratios, offset margins, overlapping elements.
- 8-10: Full asymmetry, bold whitespace, unconventional structure.

**MOTION_INTENSITY** — Animation level.
- 1-3: Hover and active states only. No auto-animations.
- 4-7: Smooth transitions, staggered load-ins, scroll reveals.
- 8-10: Magnetic cursors, spring physics, parallax, scroll-driven choreography.

**VISUAL_DENSITY** — Content per screen.
- 1-3: Spacious. One focal point at a time. Premium feel.
- 4-7: Standard web app spacing.
- 8-10: Packed tight. Monospace numbers, 1px dividers, dashboard mode.
